# Konnect Frontend - Authentication Integration\n\n## Overview\n\nThis Next.js frontend integrates with the Konnect authentication system, providing a complete user authentication flow with automatic token refresh and secure session management.\n\n## 🛡️ Authentication Features\n\n### **Two-Factor Authentication Flow**\n- **Registration**: User creates account → Email verification via OTP\n- **Login**: User signs in → Email verification via OTP\n- Both flows converge at `/verify-otp` for secure token issuance\n\n### **Token Management**\n- **Access Token**: 15-minute lifespan, stored in memory/sessionStorage\n- **Refresh Token**: 7-day lifespan, stored in httpOnly cookie\n- **Auto-refresh**: Seamless token renewal on expiration\n- **Secure logout**: Invalidates tokens both client and server-side\n\n### **Security Features**\n- 🔒 HttpOnly cookies for refresh tokens (XSS protection)\n- 🔄 Automatic token refresh with fallback\n- 🛡️ Protected routes with auth guards\n- 🚫 Secure logout that clears all session data\n- ⏰ OTP rate limiting (5 attempts max)\n\n## 📁 Frontend Structure\n\n```\nfrontend/\n├── app/\n│   ├── login/page.js          # Login form\n│   ├── register/page.js       # Registration form  \n│   ├── verify-otp/page.js     # OTP verification\n│   └── dashboard/page.js      # Protected dashboard\n├── lib/\n│   └── api.js                 # API client with auth handling\n├── components/\n│   └── ProtectedRoute.js      # Auth guard wrapper\n├── hooks/\n│   └── useAuth.js             # Authentication hook\n└── .env.local                 # Environment configuration\n```\n\n## 🔄 Authentication Flow\n\n### **Registration Flow**\n```\n1. User fills registration form → POST /auth/register\n2. System validates & sends OTP → Redirects to /verify-otp\n3. User enters OTP → POST /auth/verify-otp\n4. System issues tokens → Redirects to /dashboard\n```\n\n### **Login Flow**\n```\n1. User enters credentials → POST /auth/login\n2. System validates & sends OTP → Redirects to /verify-otp\n3. User enters OTP → POST /auth/verify-otp\n4. System issues tokens → Redirects to /dashboard\n```\n\n### **Token Refresh Flow**\n```\n1. Access token expires (15min) → API returns 401\n2. Frontend auto-calls → POST /auth/refresh-token\n3. System validates refresh cookie → Issues new access token\n4. Request retries with new token → Success\n```\n\n## 🚀 Getting Started\n\n### **1. Install Dependencies**\n```bash\ncd frontend\nnpm install\n```\n\n### **2. Environment Setup**\nThe `.env.local` file is already configured:\n```env\nNEXT_PUBLIC_API_URL=http://localhost\nNODE_ENV=development\n```\n\n### **3. Start Development Server**\n```bash\nnpm run dev\n```\n\n### **4. Access the Application**\n- Frontend: http://localhost:3000\n- Registration: http://localhost:3000/register\n- Login: http://localhost:3000/login\n- **API Gateway**: http://localhost/auth (via Traefik)\n- **Traefik Dashboard**: http://localhost:8080\n\n## 🔧 API Integration\n\n### **Authentication Methods**\n\nThe `lib/api.js` provides these methods:\n\n```javascript\n// Basic auth requests\nawait authPost('/auth/register', { name, email, password });\nawait authPost('/auth/login', { email, password });\nawait authPost('/auth/verify-otp', { email, otp });\n\n// Protected requests (auto-refresh on 401)\nawait authenticatedRequest('/api/protected-endpoint');\n\n// Token management\nsetAccessToken(token);     // Store access token\ngetAccessToken();          // Retrieve access token\nisAuthenticated();         // Check auth status\nclearAuth();              // Clear all auth data\nlogout();                 // Full logout\n```\n\n### **Protected Routes**\n\nWrap sensitive pages with `ProtectedRoute`:\n\n```jsx\nimport ProtectedRoute from '@/components/ProtectedRoute';\n\nexport default function DashboardPage() {\n  return (\n    <ProtectedRoute>\n      {/* Your protected content */}\n    </ProtectedRoute>\n  );\n}\n```\n\n### **Auth Hook Usage**\n\n```jsx\nimport { useAuth } from '@/hooks/useAuth';\n\nfunction MyComponent() {\n  const { authenticated, loading, logout } = useAuth();\n  \n  if (loading) return <div>Loading...</div>;\n  \n  return (\n    <div>\n      {authenticated ? (\n        <button onClick={logout}>Logout</button>\n      ) : (\n        <Link href=\"/login\">Login</Link>\n      )}\n    </div>\n  );\n}\n```\n\n## ⚠️ Prerequisites\n\n**Ensure these services are running:**\n\n1. **Auth Service** (Port 3002)\n   ```bash\n   cd Konnect/services/auth-service\n   npm run dev\n   ```\n\n2. **Email Service** (Background)\n   ```bash\n   cd Konnect/services/email-service  \n   npm run dev\n   ```\n\n3. **Redis** (For OTP storage)\n4. **RabbitMQ** (For email queue)\n5. **MongoDB** (For user data)\n\n## 🎯 Key Features\n\n### **Automatic Token Refresh**\n- No user interruption when access tokens expire\n- Fallback to login when refresh tokens expire\n- Handles network failures gracefully\n\n### **Enhanced UX**\n- Loading states for all async operations\n- Clear error messages and validation\n- Visual feedback for authentication status\n- Mobile-friendly OTP input\n\n### **Security Best Practices**\n- Tokens never exposed in URLs or logs\n- Refresh tokens in httpOnly cookies only\n- Secure logout clears all session data\n- Protected routes prevent unauthorized access\n\n### **Development Features**\n- Environment-based configuration\n- Error boundaries for better debugging\n- Console logging for development\n- Token status display in dashboard\n\n## 🔍 Testing Authentication\n\n1. **Register New User**:\n   - Navigate to `/register`\n   - Fill form and submit\n   - Check email for OTP\n   - Verify OTP → Should reach dashboard\n\n2. **Login Existing User**:\n   - Navigate to `/login` \n   - Enter credentials\n   - Check email for OTP\n   - Verify OTP → Should reach dashboard\n\n3. **Token Refresh** (automatic):\n   - Stay logged in for 15+ minutes\n   - Make any protected request\n   - Should auto-refresh token seamlessly\n\n4. **Logout**:\n   - Click logout in dashboard\n   - Should clear tokens and redirect to login\n\n## 🐛 Troubleshooting\n\n### **\"Cannot reach authentication service\"**\n- Verify auth service is running on port 3002\n- Check NEXT_PUBLIC_API_URL in `.env.local`\n- Ensure no CORS issues\n\n### **\"OTP verification failed\"**\n- Check email service is running and consuming queue\n- Verify Redis connection for OTP storage\n- Check spam folder for OTP emails\n\n### **\"Authentication expired\"**\n- Both access and refresh tokens have expired\n- User needs to log in again\n- Check if auth service is properly handling refresh tokens\n\n### **Infinite redirect loops**\n- Clear browser storage: `sessionStorage.clear()`\n- Check for circular auth guard logic\n- Verify token validation logic\n\nThis frontend provides a production-ready authentication experience with robust token management and security best practices! 🎉
